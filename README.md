# ピアノ音当てゲーム

ピアノで鳴った音を聴き、画面上の鍵盤または接続したMIDIキーボードを使って正解を当てる、ゲーム性の高い耳トレアプリです。

## 起動方法

ローカルHTTPサーバー経由で開きます。`file://` で直接開くと、音源やMIDIの利用条件を満たせません。

```
python -m http.server 8000 --bind 127.0.0.1
```

起動後、`http://127.0.0.1:8000/` を開きます。リポジトリに含まれる `start-piano-ear-game.bat` は、専用のローカルHTTPサーバーを `8128` 番ポートで起動するランチャーです。MIDIを使う場合は、`file://` ではなくlocalhostまたはHTTPSで開いてください。

**注意**: MIDIキーボードを使用する場合は、`file://` ではなく `http://localhost` または `https` で開いてください。

## GitHub Pages への公開

このフォルダをリポジトリのルートにして公開できます。`index.html`、`style.css`、`app.js`、`assets/`、`licenses/` をコミットし、GitHubの **Settings > Pages** で対象ブランチのルート（`/`）を公開元に選択してください。すべてのアプリ内参照は相対パスのため、プロジェクトページ（`https://<user>.github.io/<repo>/`）でも動作します。

`assets/audio/bgm/*.mp3` は各ファイルがGitHubの100 MiB上限を超えるため、`.gitignore` で除外しています。BGMがない場合でもゲームは無音で継続します。BGMを公開する場合は、100 MiB未満のWeb向けファイルへ差し替えるか、外部ホスティングを利用してください。

## BGMの配置

`assets/audio/bgm/` にBGMアセットが存在する場合、ホーム・モード選択・カレンダー画面で再生します。BGMファイルを読み込めない場合は設定画面にエラー状態を表示し、「再読み込み」「ファイルを再選択」「BGMなしで続ける」を選べます。BGMがなくてもゲーム本体は無音で継続できます。

## ピアノ音源の配置

`assets/audio/piano/` にSalamander Grand Piano V3のFLACサンプルを34個配置しています。作者は Alexander Holm、ライセンスは CC BY 3.0 です。C2〜C6の17アンカー音×velocity layer 2（v5/v12）で、出題範囲全体をカバーします。配布元は [SalamanderGrandPiano リポジトリ](https://github.com/sfzinstruments/SalamanderGrandPiano) です。

ファイル名ルール：
- 半角英数字を使用
- シャープ記号は `s` で表現
  - 例: C♯4 → `piano_Cs4_v5.flac`, F♯3 → `piano_Fs3_v5.flac`
- オクターブ番号付き・velocity layer付き
  - 例: `piano_C4_v5.flac`, `piano_Ds4_v12.flac`

範囲：C2〜C6（MIDI 36〜84）。読み込み時に全34ファイルをHTTP 200、`decodeAudioData`、AudioBufferキャッシュまで検証します。

音源の出典・ライセンスは [`licenses/audio/SalamanderGrandPiano-CC-BY-3.0.txt`](licenses/audio/SalamanderGrandPiano-CC-BY-3.0.txt) に記載しています。

ピアノ音源のOscillatorNodeフォールバックは使用しません。1件でも失敗した場合はエラー状態にしてセッション開始を無効化し、失敗ファイル・HTTPステータス・デコード理由と再読み込みボタンを表示します。

## 効果音の配置

`assets/audio/sfx/` に効果音アセットを配置できます。音源が利用できない場合でも、画面表示とゲーム進行は継続します。

設定画面では「効果音を使用」をON/OFFでき、効果音量は0〜100%で調整できます。全体ミュート、プロフィール別設定、バックアップ・復元、リロード後の設定復元に対応します。ピアノ音量は効果音量とは独立して保存されます。

## MIDI利用条件

- ブラウザが Web MIDI API に対応していること
- `file://` ではなく `http://localhost` または `https` で開くこと
- USB または Bluetooth MIDI キーボードが接続されていること

MIDI非対応環境では、画面のピアノ鍵盤を使用してすべてのゲームモードをプレイできます。

MIDI velocity（入力のvelocity）はピアノサンプルのvelocity layer選択と再生レベルに反映され、note-offでは再生中ノートを停止します。

## 結果画面からの間違い問題リトライ

セッション結果画面の「間違えた問題を復習」は、復習一覧を経由せず、直前セッションの不正解問題だけを出題順のまま収録した新しい復習セッションを直接開始します。1問だけの復習にも対応し、復習セッションで再び間違えた問題は同じ操作でさらに新しい復習セッションとして繰り返し復習できます。

復習セッションは通常セッションと同じ保存・途中再開・履歴・セッション詳細・分析・ヒートマップ・バックアップの経路を使用します。元セッションは変更せず、`sourceSessionId` と問題ごとの識別子を保持します。正解情報や問題固有の再生情報は保持し、過去の回答・正誤・回答時間・一時選択状態は初期化します。BGM、効果音、ピアノ音源のready確認、音源エラー時の開始不能は通常セッションと同じです。

全問正解の結果では、復習対象がないためボタンを表示せず、「全問正解です。復習する間違いはありません。」と表示します。

不正解後は演出が終わるまで入力をロックし、次の問題を自動開始しません。演出完了後に「次の問題へ」を表示し、最終問題では「結果を見る」を表示します。途中リロード後も回答済み状態と進行ボタンを復元します。

セッション画面はスマートフォンとPCの幅に合わせて鍵盤・問題表示・操作ボタンを再配置し、鍵盤ラベルの折り返しと黒鍵ラベルの重なりを抑えています。クリーム／ダークテーマでも音名を読みやすく表示します。

## データ保存

- プロフィール設定、セッション履歴、チェックイン、問題プール、復習・分析用データはブラウザー内の `IndexedDB`（`PianoEarGameDB`）に保存されます。
- 現在プロフィール、テーマ、音声有効状態、未完了セッションの補助情報など、アプリ専用の `pe_` 接頭辞の値は `localStorage` に保存されます。
- ブラウザーのサイトデータ、IndexedDB、localStorageを削除すると、保存データが消える可能性があります。
- 設定画面の「データ管理画面を開く」から、正式なJSONバックアップの書き出しと復元ができます。

## 音源読み込みエラー対策

- ホーム、設定、データ管理、履歴などの画面は音源エラー時も利用できます
- 音源がloading中はセッションを開始できません
- error時は電子音へ切り替えず、失敗ファイル・HTTPステータス・デコード理由を表示します
- 「再読み込み」で同じファイルを二重ロードせず再試行できます

## 既知の制限

- MIDI は `file://` プロトコルでは使用できません
- iOS Safari では自動再生が制限されるため、初回に「音声を有効化」ボタンが必要です
- 一部のブラウザでは IndexedDB の保存容量に制限があります
- **BGMのバックアップ範囲**: JSONには設定値やアプリ専用localStorageの値を含めますが、BGM・ピアノ音源・効果音のバイナリ、Blob URL、外部アセットそのものは含めません。ユーザーが選択したBGMはメモリ上の一時読み込みです。
- セッション開始時はBGMを一時停止し、出題・回答中は再生しません。セッション終了後は開始前に再生中だった場合だけ、再生位置を維持して再開します。開始前に停止中だった場合は勝手に再開しません。
- 自動再生制限のため、初回の音声有効化またはユーザー操作後にAudioContextを再開します。
- 履歴のセッション詳細からホームへ移動したときは、詳細のsessionIdと戻り先状態をクリアし、遅延した詳細描画でホームへ戻らないようにします。

## 第12段階：バックアップと復元

### バックアップ

設定画面の「データ管理画面を開く」から、次のボタンを使います。

- 「全プロフィールをJSON保存」: IndexedDBの全プロフィールと関連データ、アプリ専用localStorageをUTF-8 JSONとして保存します。
- 「現在のプロフィールだけJSON保存」: 現在のプロフィール、そのプロフィールに属するセッション、チェックイン、問題プールなどを保存します。別プロフィールのIndexedDBレコードは含めません。

JSONには `backupVersion`、`schemaVersion`、書き出し範囲、manifest、データ本体が含まれます。BGMやその他音源のバイナリはバックアップ対象外です。

### 復元

「バックアップJSONを選択」でファイルを読み込むと、復元前にバージョン、構造、参照、値の範囲、安全な文字列形式を検証し、エラー一覧または復元プレビューを表示します。検証とプレビューが完了するまで現在のデータは変更しません。

- 「現在のデータへ追加」: 現在のデータを保持し、同一レコードをスキップします。IDが衝突するレコードは参照を保ったまま新しいIDへ再発行します。
- 「現在のデータを置き換え」: 検証済みバックアップでデータを置き換えます。置き換え前に確認ダイアログが表示されます。
- 「新しいプロフィールとして復元」: 選択プロフィールを新しいIDへ複製し、元のプロフィールとそのデータは保持します。

未対応の `backupVersion` または `schemaVersion`、構造不正・参照切れ・範囲外の値・不正JSONは拒否します。バックアップJSONを手作業で編集したり、別アプリのJSONを読み込んだりしないでください。

### 初期化と削除

- 「現在プロフィールの設定だけ初期化」: 現在プロフィールの設定を既定値へ戻し、履歴とチェックインを保持します。
- 「現在プロフィールの履歴だけ削除」: 現在プロフィールの完了履歴、回答履歴、復習候補、分析・ヒートマップ集計を削除します。設定、チェックイン、問題プール、別プロフィールは保持します。
- 「現在プロフィールを初期化」: 現在プロフィールの設定、履歴、チェックイン、問題プール、復習・分析データを初期化します。プロフィール名とアイコンは保持します。
- 「全データ削除（2段階確認）」: 全プロフィール、履歴、設定、チェックイン、問題プール、復習・分析データを削除し、初期プロフィールを作成します。第1確認の後、確認文字列「全データ削除」の入力が必要です。無関係なlocalStorageは保持します。

### エラー時の対処

- ピアノ音源が読み込めない場合は無音のまま進行せず、セッション開始を停止して「再読み込み」を使用します。電子音フォールバックはありません。
- BGMが読み込めない場合は設定画面の再読み込み・再選択・BGMなしを利用できます。
- MIDI非対応、権限拒否、機器未接続の場合は画面鍵盤でプレイできます。Web MIDI APIはlocalhostまたはHTTPSなどの安全なコンテキストと、ブラウザーの権限許可が必要です。
- `favicon.ico` が配置されていない環境では404が記録されることがありますが、アプリ本体の画面・音・データ管理には影響しません。
- データ破損が疑われる場合は、現在のブラウザーデータを削除せず、検証済みの全体バックアップをプレビューしてから置き換え復元してください。復元できない場合は、別の正常なバックアップを使用してください。
